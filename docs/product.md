# Product

## 产品定位

本项目用于把私有 Markdown 文档渲染成可公开访问的图片 URL，主要面向“论坛禁止编辑正文但允许贴外部图片”的发布场景。

产品目标不是一次性导出图片文件，而是让用户持续维护文档内容，并让外部论坛始终通过同一个图片地址访问该文档当前版本的图片。

## 目标用户

- 需要在论坛、社区或不支持富文本编辑的平台发布内容的用户
- 希望用 Markdown 持续维护内容，同时对外以图片形式展示的用户
- 更关注 PC 端阅读体验，但仍希望手机端可读的用户

## 核心价值

- 用 Markdown 编写内容，而不是直接在论坛里排版
- 通过固定公开 URL 对外发布图片，减少重复发帖和重复上传
- 文档内容更新后，图片内容同步更新，外链地址保持不变
- 私有文档与公开图片分离，便于作者管理内容与发布状态

## 核心概念

### 文档

- 文档是私有对象
- 查看和编辑文档需要登录
- 文档包含 Markdown 内容和发布配置
- 文档是图片的唯一来源

### 公开图片

- 每个文档绑定一张公开图片
- 公开图片使用固定 UUID 作为访问标识
- UUID 对应的是“当前公开图片槽位”，不是某次历史快照
- 外部论坛只访问公开图片，不访问文档本身

## 产品原则

- 一份文档只对应一张公开图片
- 图片 URL 固定不变
- 修改文档后，更新同一 URL 下的图片内容
- 不保留历史图片版本
- 不支持分页图片
- 从未生成、已删除、已过期统一返回 `404`

## 典型使用流程

### 1. 创建文档

用户创建一个私有 Markdown 文档，并设置：

- Markdown 内容
- 阅读密度预设
- 图片是否允许过期

创建后，系统为该文档分配一个固定的公开图片 UUID。

### 2. 首次生成图片

用户触发生成后：

- 系统渲染该文档对应的当前图片
- 公开图片 URL 开始可访问
- 用户可将该 URL 粘贴到外部论坛中使用

### 3. 编辑文档

当用户修改文档内容后：

- 文档仍保持私有
- 系统更新同一 UUID URL 下的图片内容
- 外部论坛引用地址不变

### 4. 删除或过期

当图片被删除或过期时：

- 公开 URL 返回 `404`
- 文档本身不删除
- 用户仍可登录查看和编辑文档
- 后续可重新生成图片，并继续使用同一个 UUID URL

## 阅读体验策略

### PC 优先

- 产品整体以 PC 端阅读体验为主
- 正文排版、标题层级和代码块可读性优先为桌面阅读优化

### 兼顾手机端

- 不单独为手机端生成另一套图片
- 要求手机端在缩放后仍保持基本可读
- 当内容过长时，需要提醒用户可能影响手机端阅读体验

## 阅读密度预设

系统不开放大量自由调节参数，而是提供少量预制阅读密度档位。

当前计划提供 5 档：

- 极疏
- 稀疏
- 标准
- 紧凑
- 极紧凑

这些档位用于控制整套版式风格，包括：

- 正文字号
- 行高
- 段落间距
- 标题间距
- 页面边距
- 代码块间距

阅读密度是文档属性：

- 创建文档时确定初始档位
- 后续允许修改

## 长图策略

- 当前产品只生成单张图片
- 不支持分页切图
- 允许生成长图
- 当内容明显过长时，系统应提供软提醒
- 软提醒只提示风险，不阻止继续生成

## 公开访问语义

公开 URL 的行为应尽量简单：

- 当前有可访问图片：返回图片
- 从未生成：`404`
- 已删除：`404`
- 已过期：`404`

对外不区分图片不存在的原因。

## 生命周期说明

### 文档生命周期

- 文档长期存在于系统内
- 文档是否存在，不由公开图片状态决定
- 删除图片不等于删除文档

### 图片生命周期

- 图片是文档的当前公开产物
- 图片可以生成、更新、删除、过期
- 图片删除或过期后可以再次生成
- 再次生成时继续使用同一个 UUID URL

## 当前非目标

以下内容不在当前产品范围内：

- 基于内容哈希的内容寻址
- 多图分页导出
- 历史图片版本保留
- 通过公开 URL 反查渲染参数
- 面向论坛平台限制的宽高、尺寸、文件大小适配
- 缓存策略设计

## 一句话总结

这是一个“私有 Markdown 文档 + 固定公开图片 URL”的发布系统：文档持续可编辑，图片持续可更新，外部引用地址保持不变。
